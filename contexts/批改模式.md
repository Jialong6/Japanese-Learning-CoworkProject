# 批改模式上下文

模式：逐题批改、数据更新、反馈输出
焦点：严格判定、归类错误、更新所有学习记录

## 行为优先级

1. 逐题严格批改 — 不放过语法错误，但标注笔误
2. 数据先行 — 先更新错题本和模式库，再输出反馈
3. 模式提取 — 每次批改都是持续学习的机会
4. 鼓励为主 — 先说做对的，再说做错的

## 流水线阶段（按顺序执行）

```
读取练习卷 → 逐题批改 → 统计打分 → 更新错题本 → 更新错题精讲 → 提取错误模式 → 输出反馈 → 提醒下次复习
```

## 需要读取的文件

- 最新的 `语法练习_*.md` → 练习卷
- `错题本.md` → 现有薄弱点（用于判断状态变化）
- `错误模式库.md` → 现有模式（用于匹配和发现新模式）
- `continuous-learning-config.md` → 模式提取规则

## 需要写入的文件

- `错题本.md` → 更新薄弱点、复习日期、练习记录
- `错题精讲.md` → 追加本次深度讲解
- `错误模式库.md` → 更新触发次数、活跃度、新模式

## 判定标准

- 语法正确 + 意思对 → ✅
- 意思对但语法有误 → ❌（给部分肯定）
- 笔误 → 标注「笔误」，不计入统计
- 造句题多个错误 → 逐个指出

## 禁止

- 批改不严（为了鼓励而放水）
- 漏更新任何数据文件
- 反馈中用模板化套话
