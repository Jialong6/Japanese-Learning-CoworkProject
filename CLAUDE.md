## 出题规则

### 题型菜单（每次练习从中选择 5-8 种）

1. **动词变形** — ない形、原形、て形、た形、ます形互转
2. **形容词/名词变形** — て形、过去式、否定式
3. **填助词** — は/が/を/に/で/へ/と/から/まで/も 等
4. **选择题** — 语法点辨析（句型、副词、助词等）
5. **用法辨析** — 如「を」的宾语 vs 离经用法、まで vs までに 等
6. **造句/翻译** — 中译日，综合运用语法和词汇
7. **纠错题** — 找出并改正句中语法错误
8. **词汇辨析** — 近义词、易混词选择
9. **阅读理解** — 短文阅读回答问题
10. **排列组词** — 打乱的词重新排列成正确句子

### 出题原则

- 每次练习 60-80 小题，耗时约 30-45 分钟
- **70% 考察薄弱点**（参考错题本中的薄弱语法点和错题频率）
- **20% 巩固已掌握内容**
- **10% 适度拓展**（用已学语法的新组合或稍复杂的句子）
- 造句/翻译题优先使用贴近生活的场景
- 练习卷文件命名格式：`语法练习_YYYY-MM-DD_范围.md`

## 艾宾浩斯复习计划

### 复习间隔

新学/新错的语法点，按以下间隔安排复习：

| 轮次 | 间隔 | 说明 |
|------|------|------|
| 第1次 | 当天 | 出错后立即在反馈中讲解 |
| 第2次 | 1天后 | 下一次练习重点考察 |
| 第3次 | 3天后 | 再次出现在练习中 |
| 第4次 | 7天后 | 一周后复习 |
| 第5次 | 14天后 | 两周后复习 |
| 第6次 | 30天后 | 一个月后复习，通过则标记为"已掌握" |

### 状态定义

- **🔴 未掌握**：错误2次及以上，或最近一次练习仍错
- **🟡 不稳定**：错过但最近一次做对了
- **🟢 已掌握**：连续2次以上做对，且间隔超过7天

## 批改规则

每次 Jay 做完练习并告知后：

1. **逐题批改** — 对每题标注 ✅ 或 ❌，错题给出正确答案和简明解释
2. **打分** — 统计总分和各题型得分率
3. **错题归类** — 按语法点分类，记录到 `错题本.md`
4. **薄弱点分析** — 总结这次暴露出的主要问题，给出针对性建议
5. **更新复习计划** — 在错题本中更新下次复习时间和重点

（归纳共性问题）

（重点复习什么、建议什么时候做下一次练习）

## 交互约定（Commands 系统）

每个命令对应 `commands/` 目录下的一个 `.md` 文件，包含详细的执行流程。收到触发词时，先读取对应 command 文件，再按流程执行。

| 触发词 | Command 文件 | 说明 |
|--------|-------------|------|
| `出题` / `给我出练习` | `commands/出题.md` | 根据错题本和复习计划生成新练习卷 |
| `做完了` / `批改` | `commands/批改.md` | 逐题批改、更新错题本和错题精讲 |
| `复习计划` / `今天练什么` | `commands/复习计划.md` | 按艾宾浩斯间隔给出复习安排 |
| `看看错题本` / `我的薄弱点` | `commands/薄弱点.md` | 汇报错题统计和薄弱点分析 |
| `更新词汇量到第X课` / `我学到第X课了` | `commands/更新进度.md` | 更新词汇量和语法范围 |
| 提供新的语法笔记文件 | — | 读取并纳入出题范围 |

> **执行原则：** 收到触发词后，先读取对应的 command 文件获取完整流程，再严格按步骤执行。不凭记忆执行，每次都读文件。

## Git 自动版本管理

### 自动 commit（每次文件改动后）

每当对项目文件执行写入或编辑操作后，立即执行 git commit。规则：

1. **触发时机：** 每次用 Write/Edit 工具修改了项目文件后，在回复用户之前先 commit
2. **commit 范围：** 只 add 刚刚改动的文件，不用 `git add -A`
3. **commit message 格式：** 用中文，简明说明改了哪个文件、做了什么
   - 例：`更新 错题本.md：添加 2026-04-02 练习记录，新增3个薄弱语法点`
   - 例：`新建 语法练习_2026-04-02_16-19课.md：生成60题复习练习卷`
   - 例：`更新 CLAUDE.md：词汇范围从第25课扩展到第30课`
4. **批量操作：** 如果一个命令流程中连续改了多个文件（如批改时更新错题本+错题精讲+错误模式库），可以合并为一次 commit，message 中列出所有改动
5. **不 commit 的情况：** 临时文件、.DS_Store、用户还没做完题的练习卷

### 每次任务完成后 push

每次 Jay 发消息、Claude 完成对应任务后，在回复用户之前执行：

1. 确保所有改动已 commit（上一步的自动 commit 应该已覆盖）
2. 执行 `git push origin main`
3. 如果 push 失败（认证问题等），告诉 Jay 具体错误，让 Jay 协助解决
4. push 成功后简要告知
