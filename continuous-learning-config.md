# 持续学习配置

> 定义错误模式提取规则。批改命令执行第6步（提取错误模式）时参考此文件。

---

## 模式检测规则

### 值得记录的模式（DETECT）

| 类别 | 检测条件 | 示例 |
|------|---------|------|
| **母语干扰** | 错误源于中文/韩语语法习惯的直接迁移 | 中文"在"→へ、中文语序直译 |
| **规则过度泛化** | 把一般规则套用到特殊情况 | 不规则动词用一般变形 |
| **概念混淆** | 两个近似语法点/词汇长期分不清 | もう vs また、で vs に |
| **结构性缺陷** | 造句/翻译时暴露的系统性思维问题 | 不拆句子结构直接翻译 |
| **记忆盲区** | 某个规则学过但反复遗忘 | 拨音便规则、て形连接 |

### 不值得记录的模式（IGNORE）

| 类别 | 原因 |
|------|------|
| **单纯笔误** | 手误、少写一个假名，不是思维问题 |
| **一次性粗心** | 做对过多次，偶尔马虎一次 |
| **题目理解偏差** | 因为没看清题目要求而错，不是语法问题 |
| **超纲内容** | 涉及未学范围的错误 |

## 模式生命周期

```
新发现 → ⚡ 偶尔出现 → 🔥 高频出现 → （持续做对后）→ 💤 已消退 → （删除/归档）
```

### 升级条件

| 变化 | 条件 |
|------|------|
| ⚡ → 🔥 | 连续2次练习都触发该模式，或累计触发次数 ≥ 3 |
| 🔥 → ⚡ | 最近2次练习都没触发 |
| ⚡ → 💤 | 最近触发距今 ≥ 14天 且 最近2次练习没触发 |
| 💤 → 移除 | 距最近触发 ≥ 30天，可从总览表移除（保留在更新日志中作为历史记录） |

### 复燃规则

已消退（💤）的模式如果再次触发：
- 立即升级回 ⚡
- 如果是在复燃后的下一次练习中又触发 → 直接升级到 🔥
- 在错题精讲中特别标注"旧模式复燃"

## 模式详情模板

新模式必须包含以下字段：

```markdown
### PXX｜模式名称

**活跃度：** ⚡/🔥/💤
**触发次数：** X（列出关联题号）
**首次发现：** YYYY-MM-DD
**最近触发：** YYYY-MM-DD
**思维路径：** Jay 犯这个错时脑子里在想什么
**实际规则：** 正确的日语规则
**检测信号：** 什么场景/题型下容易触发这个模式
**针对性出题：** 出什么样的题来针对这个模式
**韩语类比：** （可选）韩语中有没有类似的点可以帮助记忆
```

## 出题联动规则

| 模式活跃度 | 出题要求 |
|-----------|---------|
| 🔥 高频 | 每次练习**至少2题**直接测试，且题目设计要命中「检测信号」所描述的场景 |
| ⚡ 偶尔 | 每次练习**至少1题**，可以是间接考察 |
| 💤 已消退 | 每2-3次练习出1题，确认是否真的消退 |

---

## 统计指标

批改完成后，在更新日志中记录以下指标：

```markdown
| 日期 | 练习 | 新增模式 | 更新模式 | 消退模式 | 活跃🔥数 | 总模式数 | 模式命中率 |
```

**模式命中率** = 本次错题中能归入已有模式的错题数 / 本次总错题数
- 命中率高（>70%）→ 说明模式库准确，Jay 的问题是"老毛病反复犯"
- 命中率低（<30%）→ 说明出现了新类型的错误，需要关注
