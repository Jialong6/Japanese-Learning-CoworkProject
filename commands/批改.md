# 批改 — 批改练习卷并更新学习记录

触发词：`做完了`、`批改`、`改卷`

> **设计理念：** 借鉴 Orchestrate 流水线思路，批改流程分为 6 个阶段，
> 每个阶段的输出成为下一个阶段的输入。先完成所有数据处理，最后再输出反馈。

## 上下文

执行前先读取 `contexts/批改模式.md`，切换到批改模式。

---

## 流水线总览

```
阶段1：读取    →  阶段2：批改    →  阶段3：统计
  ↓                                       ↓
阶段6：反馈  ←  阶段5：模式提取  ←  阶段4：数据更新
```

---

## 阶段1：读取（INPUT）

**目标：** 收集所有需要的数据

1. 找到最新的 `语法练习_*.md` 文件
2. 如果有多份未批改的练习卷，问 Jay 批改哪一份
3. 读取练习卷全文，确认 Jay 已经填写了答案
4. 读取 `错题本.md` → 获取已有薄弱语法点和状态
5. 读取 `错误模式库.md` → 获取已有模式列表
6. 读取 `continuous-learning-config.md` → 获取模式提取规则

**阶段产出（交接到阶段2）：**
- 练习卷内容 + Jay 的答案
- 已有薄弱点列表
- 已有模式列表

---

## 阶段2：逐题批改（GRADE）

**目标：** 对每题做出判定

对每一道题：

1. 判定对错，标注 ✅ 或 ❌
2. 错题必须给出：
   - 正确答案
   - 简明解释（1-2句话说清为什么错）
   - 涉及的语法点
3. 区分笔误和语法错误：
   - 笔误（打字错误、少一个假名等）→ 标注「笔误」，不计入错题统计
   - 语法错误 → 正常计入
4. 造句/翻译题的评分标准：
   - 语法正确、意思对 → ✅
   - 意思对但有语法错误 → ❌ 但给部分肯定
   - 多个错误的 → 逐个指出

**阶段产出（交接到阶段3）：**
- 每题的判定结果（✅/❌/笔误）
- 错题列表（题号、错误类型、涉及语法点、Jay的错误答案、正确答案）

---

## 阶段3：统计打分（SCORE）

**目标：** 量化本次表现

1. 计算总分（排除笔误后的有效题数为分母）
2. 按题型统计得分率
3. 按语法点统计得分率
4. 与历次练习对比（如果有历史数据）

```
总分：XX/XX（XX%）

| 题型 | 得分 | 得分率 | 与上次对比 |
|------|------|--------|-----------|
| ... | .../... | ...% | ↑/↓/→ |
```

**阶段产出（交接到阶段4）：**
- 得分统计表
- 错题清单（按语法点归类）
- 各题型得分率

---

## 阶段4：数据更新（UPDATE）

**目标：** 更新错题本和错题精讲

### 4A. 更新错题本（`错题本.md`）

1. **新错的语法点**（之前没错过）：
   - 添加到薄弱语法点汇总表
   - 状态设为 🔴 未掌握
   - 错误次数 = 1
   - 下次复习 = 明天

2. **再次做错的语法点**（之前错过）：
   - 错误次数 +1
   - 状态保持 🔴 未掌握
   - 下次复习重置为明天

3. **之前错过但这次做对的**：
   - 状态更新为 🟡 不稳定
   - 按艾宾浩斯间隔推进下次复习日期

4. **连续2次以上做对且间隔>7天的**：
   - 状态更新为 🟢 已掌握

5. 在练习记录区追加本次记录

### 4B. 更新错题精讲（`错题精讲.md`）

追加本次错题的深度讲解：
- 按「事实驱动」风格写（短句、可验证事实、不用模板连接词）
- 每道错题：错在哪 → 规则是什么 → 对比正误 → 变式练习
- 如果有韩语类比能帮助理解的，加上

**阶段产出（交接到阶段5）：**
- 更新后的薄弱点状态
- 本次新增/变化的语法点列表
- 错题精讲内容

---

## 阶段5：模式提取（LEARN）

**目标：** 持续学习 — 从本次错题中提取和更新错误模式

> 参考 `continuous-learning-config.md` 中的规则执行

### 5A. 匹配已有模式

逐一检查本次错题，是否命中已有模式（P01-PXX）？
- 命中 → 触发次数 +1，更新「最近触发」日期
- 升级判定：参考 config 中的升级条件
- 降级判定：参考 config 中的降级条件

### 5B. 发现新模式

本次错题中是否有不属于任何已有模式的新思维路径？
- 先用 config 中的 DETECT 规则判断是否值得记录
- 再用 IGNORE 规则排除不值得记录的
- 创建新模式条目（使用 config 中的模板）

### 5C. 更新统计

- 更新模式总览表
- 更新更新日志（包含模式命中率）

**阶段产出（交接到阶段6）：**
- 命中的模式列表
- 新发现的模式
- 活跃度变化
- 模式命中率

---

## 阶段6：输出反馈（REPORT）

**目标：** 向 Jay 汇报完整结果

按以下结构输出：

```markdown
## 批改结果：语法练习_XXXX

### 得分
总分：XX/XX（XX%）
各题型：...（含与上次对比）

### 错题详解
（逐题讲解 — 来自阶段2的判定和阶段4B的精讲）

### 薄弱点总结
（归纳共性问题 — 来自阶段3的语法点统计）

### 模式库动态
- 命中已有模式：PXX（名称）× N次 ...
- 新发现模式：PXX（名称）— 简要描述
- 活跃度变化：PXX 从⚡升级为🔥 ...
- 模式命中率：XX%

### 下次练习建议
（重点复习什么、建议什么时候做下一次练习）
```

### 反馈原则

- 先表扬做对的、进步的
- 错题讲解简明扼要
- 模式库动态用一段话总结，不要列太多细节
- 最后用鼓励的语气给出下次建议

---

## Git 操作

阶段4和阶段5涉及多个文件写入（错题本 + 错题精讲 + 错误模式库），
可以合并为一次 commit，message 中列出所有改动。

---

## 出错处理

- 如果练习卷中某题 Jay 没有作答（空着）→ 标注「未作答」，不计分
- 如果答案模糊（比如造句题意思对但表达不常用）→ 标注「可接受但建议改进」，算 ✅
- 如果 Jay 写了多个答案 → 取最后一个
